# Nathaniel Senyo Gemegah Portfolio (Astro + Tailwind)

This is a small, fast, and minimal portfolio site for **Nathaniel Senyo Gemegah**, built with **Astro** and **Tailwind CSS** and designed to be deployed on **Cloudflare Pages**.

All copy and project details are **placeholder content**. Replace them with your own text when you are ready.

---

## Tech Stack

- **Framework**: [Astro](https://astro.build/) (static output, zero JS by default)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/)
- **Fonts**: [Switzer](https://www.fontshare.com/fonts/switzer) (body) & [Khand](https://www.fontshare.com/fonts/khand) (headings) via Fontshare
- **Deployment target**: [Cloudflare Pages](https://pages.cloudflare.com/)

## Project Structure

- `src/pages/index.astro` – Home page with hero + primary newsletter signup

- `src/styles/globals.css` – Tailwind base styles and custom utilities
- `public/robots.txt` – Basic robots configuration (update domain)
- `public/favicon.svg` – Minimal placeholder favicon
- `astro.config.mjs` – Astro configuration (static output + sitemap + Tailwind)
- `tailwind.config.cjs` – Tailwind config (colors, fonts, content paths)

## Getting Started Locally

### 1. Install dependencies

From the project root:

```bash
npm install
```

### 2. Run the development server

```bash
npm run dev
```

Then open the URL shown in your terminal (usually `http://localhost:4321`).

### 3. Build for production

```bash
npm run build
```

This outputs a static site to the `dist/` folder.

You can preview the production build with:

```bash
npm run preview
```

## Cloudflare Pages Deployment

You can deploy this project directly to **Cloudflare Pages** without extra server configuration.

1. Push this repository to GitHub, GitLab, or Bitbucket.
2. In the Cloudflare dashboard, go to **Pages** → **Create a project**.
3. Connect your repository and choose the branch to deploy from (e.g. `main`).
4. When prompted for build settings:
   - **Framework preset**: `Astro` (or "None" and configure manually)
   - **Build command**: `npm run build`
   - **Build output directory**: `dist`
5. (Optional but recommended) Set the Node version in **Environment variables** → `NODE_VERSION` to a supported version (e.g. `18` or `20`).
6. Save and start the first build.

Once deployment finishes, Cloudflare will give you a live URL you can later replace with a custom domain.

## SEO & Configuration Notes

- Update the `site` URL in `astro.config.mjs` from `https://example.com` to your actual production domain once it is live. This affects the generated sitemap and canonical URLs.
- The sitemap is automatically generated by `@astrojs/sitemap`.
- `public/robots.txt` currently references `https://example.com/sitemap-index.xml`. Change this to your real domain after deployment.
- Each page sets its own `<title>` and `<meta name="description">` via `BaseLayout.astro`.
